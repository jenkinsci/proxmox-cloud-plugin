The login shell OpenSSH is configured to use on the Windows agent (its DefaultShell registry value). Jenkins starts the agent with a cd "<dir>" && java ... command, and this setting decides how that command is quoted for the shell.
  • Command Prompt (cmd.exe): the OpenSSH default when no DefaultShell is set. Runs the command as-is.
  • Windows PowerShell 5.x: rejects && ("not a valid statement separator in this version"), so the command is wrapped as cmd /c '<command>' and handed to cmd.exe. Select this if your agent's DefaultShell points at ...\WindowsPowerShell\v1.0\powershell.exe.
  • PowerShell 7+ (pwsh): supports && natively, so it is run as-is like Command Prompt.
Only applies to Windows agents.
