How often the periodic orphan cleanup reconciles this cloud (only relevant when Cleanup Orphaned Agents is enabled). Each pass destroys jenkins-managed VMs with no Jenkins node and removes dead agent nodes whose backing VM is gone (subject to the Orphan Cleanup Grace Period).

Changing this takes effect without a Jenkins restart, except when you reduce it below the smallest period configured when Jenkins last started: the cleanup thread is scheduled once at startup, so a reduction needs a restart to fire faster. Jenkins shows an administrative monitor warning when this happens. Increasing the period always applies immediately.

Lower it to reclaim leaked VMs and dead nodes sooner; raise it to reduce API traffic to Proxmox. Default: 600 seconds (10 minutes). Minimum: 30 seconds (also the finest cleanup cadence).
